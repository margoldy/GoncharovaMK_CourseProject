# Спецификация API

**Базовый URL:** `http://localhost:5000/api`

**Формат данных:** JSON

**Аутентификация:** JWT токен передаётся в заголовке `Authorization: Bearer <token>` и действует в течение часа


## 1. Аутентификация (`/auth`)

### 1.1 Регистрация пользователя `POST /auth/register`

Регистрация нового пользователя с ролью `pending`, который ожидает подтверждения администратором.

**Тело запроса:**
```json
{
  "login": "ivan",
  "password": "pass123",
  "full_name": "Иван Петров",
  "relation": "отец"
}
```
**Ответ:** `201 Created`
```json
{
  "id": 5,
  "login": "ivan",
  "full_name": "Иван Петров",
  "role": "pending"
}
```

**Права доступа:** открытый

---

### 1.2 Вход в систему `POST /auth/login`

**Тело запроса:**
```json
{
  "login": "admin",
  "password": "admin123"
}
```

**Ответ:** `200 OK`
```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": {
    "id": 1,
    "login": "admin",
    "full_name": "Администратор",
    "role": "admin"
  }
}
```

**Ошибка (неверные данные):** `401 Unauthorized`

**Ошибка (аккаунт не подтверждён):** `403 Forbidden`

**Права доступа:** открытый

---

## 2. Балансы (`/balances`)

### 2.1 Получить балансы счетов `GET /balances`

**Заголовок:** `Authorization: Bearer <token>`

**Ответ:** `200 OK`
```json
{
  "common": 35000, // общий счет
  "savings": 10000, // накопления
  "user_stash": 5000, // заначка текущего пользователя
  "total_stash": 5000 // общая сумма заначек всех пользователей
}
```
**Права доступа:** авторизованный пользователь

---

## 3. Члены семьи (`/members`) — только администратор

### 3.1 Список всех членов семьи `GET /members`

**Ответ:** `200 OK`
```json
[
  {
    "id": 1,
    "login": "admin",
    "full_name": "Администратор",
    "role": "admin",
    "relation": null
  },
  {
    "id": 2,
    "login": "ivan",
    "full_name": "Иван Петров",
    "role": "user",
    "relation": "отец"
  }
]
```

---

### 3.2 Создать пользователя `POST /members`

**Тело запроса:**
```json
{
  "login": "masha",
  "password": "pass123123",
  "full_name": "Мария Великая",
  "relation": "мать",
  "role": "user"
}
```

**Ответ:** `201 Created`
```json
{
  "id": 6,
  "login": "masha",
  "full_name": "Мария Великая",
  "role": "user"
}
```

---

### 3.3 Подтвердить пользователя `PUT /members/{id}/approve`

Переводит пользователя из роли `pending` в `user`.

**Ответ:** `200 OK`
```json
{
  "message": "Пользователь подтверждён, роль изменена с pending на user",
  "user_id": 2
}
```

---

### 3.4 Отклонить заявку `DELETE /members/{id}/reject`

Удаляет пользователя с ролью `pending`.

**Ответ:** `200 OK`
```json
{
  "message": "Заявка отклонена",
  "deleted_id": 3
}
```

---

### 3.5 Удалить пользователя `DELETE /members/{id}`

**Ответ:** `200 OK`
```json
{
  "deleted": 4
}
```

---

### 3.6 Назначить роль `PATCH /members/{id}/role`

**Тело запроса:**
```json
{
  "role": "admin"
}
```

**Ответ:** `200 OK`
```json
{
  "message": "Роль изменена на администратора",
  "user_id": 5,
  "new_role": "admin"
}
```

---

### 3.7 Импорт пользователей из JSON `POST /members/import`

**Тело запроса:** массив объектов

```json
[
  {
    "login": "ivan",
    "password": "pass123",
    "full_name": "Иван Петров",
    "relation": "отец",
    "role": "user"
  }
]
```

**Ответ:** `201 Created`
```json
{
  "message": "Импорт завершён: 1 успешно, 0 ошибок",
  "results": {
    "success": [...],
    "errors": []
  }
}
```

## 4. Доходы (`/incomes`)

### 4.1 Список всех доходов (админ) `GET /incomes`

**Ответ:** `200 OK`
```json
[
  {
    "id": 1,
    "member_id": 2,
    "amount": 50000,
    "income_date": "2025-06-01",
    "type_id": 1,
    "login": "ivan",
    "full_name": "Иван Петров"
  }
]
```
---

### 4.2 Список своих доходов `GET /incomes/my`

**Ответ:** `200 OK`
```json
[
  {
    "id": 1,
    "amount": 50000,
    "income_date": "2025-06-01",
    "type_id": 1
  }
]
```
---

### 4.3 Добавить доход `POST /incomes`

**Тело запроса (пользователь):**
```json
{
  "amount": 50000,
  "income_date": "2025-06-01",
  "type_id": 1
}
```

**Или тело запроса от админа, для другого пользователя:**
```json
{
  "member_id": 3,
  "amount": 30000,
  "income_date": "2025-06-01",
  "type_id": 2
}
```

**Ответ:** `201 Created`
```json
{
  "id": 2
}
```

**Автораспределение дохода:** 70% - общий счёт, 20% - накопления, 10% - заначка

**Права доступа:** пользователь (только себе), администратор (любому)

---

### 4.4 Удалить доход `DELETE /incomes/{id}`

**Ответ:** `200 OK`
```json
{
  "deleted": 2
}
```

**Права доступа:** владелец дохода или администратор

---

### 4.5 Список типов доходов (справочник) `GET /incomes/types`

## 6. Переводы (`/transfers`)

### 6.1 Список всех переводов (админ) `GET /transfers`

**Ответ:** `200 OK`
```json
[
  {
    "id": 1,
    "member_id": 2,
    "from_account_id": 3,
    "to_account_id": 1,
    "amount": 2000,
    "transfer_date": "2025-06-03",
    "full_name": "Иван Петров"
  }
]
```

---

### 6.2 Список своих переводов `GET /transfers/my`

**Ответ:** `200 OK`
```json
[
  {
    "id": 1,
    "from_account_id": 3,
    "to_account_id": 1,
    "amount": 2000,
    "transfer_date": "2025-06-03"
  }
]
```

---

### 6.3 Добавить перевод `POST /transfers`

**Тело запроса (пользователь):**
```json
{
  "from_account_id": 3,
  "to_account_id": 1,
  "amount": 2000,
  "transfer_date": "2025-06-03"
}
```

**Тело запроса (админ, для другого пользователя):**
```json
{
  "member_id": 3,
  "from_account_id": 1,
  "to_account_id": 2,
  "amount": 5000,
  "transfer_date": "2025-06-03"
}
```

**Ответ:** `201 Created`
```json
{
  "id": 2
}
```

**Ограничения для обычного пользователя:**
- Можно переводить только с заначки (3) на общий счёт (1) или накопления (2)

**Права доступа:** пользователь (только со своей заначки), администратор (любые переводы, любой пользователь)

---

### 6.4 Удалить перевод `DELETE /transfers/{id}`

**Ответ:** `200 OK`
```json

  "deleted": 2
```

**Права доступа:** только администратор
